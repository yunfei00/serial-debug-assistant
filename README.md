# Serial Debug Assistant

基于 Python 3.11、PySide6、pyserial 开发的桌面串口调试助手。

## 已实现功能

- 枚举系统串口列表
- 显示串口描述、厂商、硬件 ID 等基础信息
- 选择波特率，默认 `115200`
- 支持选择数据位、校验位、停止位
- 打开串口 / 关闭串口
- 收发解耦：接收线程与发送线程独立运行
- 发送文本（支持可选行尾：无 / CR / LF / CRLF）
- 一键发送 `AT` 命令用于快速联调
- 支持 `HEX` 发送
- 发送结果可见：状态栏会提示“发送请求已提交”与“串口已实际发送字节数”
- 实时接收并显示串口数据（修复“可发送但收不到”场景，增强读取稳定性，避免首包遗漏）
- 支持 `HEX` 显示
- 支持接收区自动换行显示
- 支持定时发送，可设置发送间隔并开始/停止
- 支持本次运行期间的发送历史记录
- 支持显示发送/接收字节统计，并可手动清零
- 支持保存接收日志到文件
- 清空接收区
- 基本异常提示，避免程序直接崩溃

## 运行方式

1. 创建并激活 Python 3.11 虚拟环境
2. 安装依赖

```bash
pip install -r requirements.txt
```

3. 启动应用

```bash
python main.py
```

## 使用说明

- 文本发送时，直接输入内容后点击“发送”，可先选择“行尾”以适配设备协议（AT 常用 `CRLF`）
- 可直接点击“发送 AT”快速下发 `AT` 命令进行链路联调
- 点击“发送”后，状态栏先显示提交结果，再显示串口线程实际发送完成的字节数
- 勾选“HEX 发送”后，发送内容请按 `01 02 0A` 这类格式输入
- 勾选“HEX 显示”后，接收区将按十六进制显示
- 打开串口前可选择波特率、数据位、校验位、停止位
- 勾选“自动换行”后，接收区会按窗口宽度换行
- 定时发送可设置毫秒间隔，然后点击“开始定时发送”
- “发送历史”会保留本次运行期间成功发送过的内容
- “发送/接收统计”会累计当前连接会话中的字节数，可手动清零
- “保存接收日志”会把当前接收区内容保存为 `UTF-8` 文本文件

## 排查“发送后收不到数据”建议

- 先确认硬件链路：
  - 自发自收测试时，需将 `TX` 与 `RX` 正确短接，并确保 `GND` 共地
  - 两个设备互连时，请交叉连接 `TX->RX`、`RX->TX`
- 确认串口参数两端一致：波特率、数据位、校验位、停止位必须完全匹配
- 若设备协议要求结尾符，请在发送内容中补上（如 `\r`、`\n` 或 `\r\n`）
- 若使用 HEX 发送，必须是合法十六进制字节串（如 `01 03 00 00 00 02 C4 0B`）
- 接收线程采用阻塞读取（基于 `read(in_waiting or 1)`），避免空轮询导致的漏读与高 CPU 占用
- 观察状态栏：
  - 出现“发送请求已提交”表示 UI 已把数据放入发送队列
  - 出现“串口已发送 X 字节”表示串口线程已实际调用底层发送

## CI/CD（自动构建与发布）

项目新增 GitHub Actions 工作流：

- 普通 `push` / `pull_request`：自动安装依赖并执行 `python -m compileall` 语法检查
- 当推送 `v*` 标签（如 `v1.0.0`）时：在检查通过后自动创建 Release，并上传源码压缩包

工作流文件：`.github/workflows/ci-release.yml`

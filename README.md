# Serial Debug Assistant

基于 Python 3.11、PySide6、pyserial 开发的桌面串口调试助手。

## 已实现功能

- 枚举系统串口列表
- 选择波特率，默认 `115200`
- 支持选择数据位、校验位、停止位
- 打开串口 / 关闭串口
- 收发解耦：接收线程与发送线程独立运行
- 发送文本（支持多行逐条发送、可选行尾：无 / CR / LF / CRLF）
- 一键发送 `AT` 命令用于快速联调
- 支持 `HEX` 发送
- 发送结果可见：状态栏会提示“发送请求已提交”与“串口已实际发送字节数”
- 实时接收并显示串口数据（修复“可发送但收不到”场景，增强读取稳定性，避免首包遗漏）
- 接收显示增加短暂聚合（约 40ms），减少 `OK` 等短报文被拆成多行
- 支持 `HEX` 显示
- 支持接收区自动换行显示
- 支持定时发送，可设置发送间隔并开始/停止
- 支持本次运行期间的发送历史记录
- 支持将发送历史按名称保存为本地命令集
- 命令集会保存为项目目录下 `data/command_sets/*.json` 文件，便于直接查看与备份
- 应用启动后默认加载上次保存/使用的命令集
- 支持通过下拉列表切换已保存的命令集
- 保存命令集后，后续发送的新命令会实时写入当前命令集文件
- 接收区按日志格式显示 send/receive 记录，包含时间戳（毫秒）
- 运行中日志会自动写入 `log/` 目录，按日期生成文件并自动追加
- 发送日志会去掉行尾换行，接收日志会去掉末尾 `\0`、首尾换行并忽略空白数据
- 点击“发送”后不再清空输入框，并可在下次启动时恢复上次发送命令
- 支持显示发送/接收字节统计，并可手动清零
- 支持保存接收日志到文件
- 清空接收区
- 支持“自动检测串口并重连”：串口掉线后每 5 秒检测一次，恢复后自动重连，掉线期间每分钟在日志区提示一次
- 发送区支持“命令描述”输入，可为每条发送命令添加说明
- 发送历史支持显示描述：格式为 `命令 (描述)`，无描述时仅显示命令
- 基本异常提示，避免程序直接崩溃

## 运行方式

1. 创建并激活 Python 3.11 虚拟环境
2. 安装依赖

```bash
pip install -r requirements.txt
```

3. 启动应用

```bash
python main.py
```

## 使用说明

- 文本发送时，输入框支持多行内容；点击“发送”会按“每行一条”逐条发送，命令间固定间隔 `0.2` 秒（并可附加行尾）
- 可直接点击“发送 AT”快速下发 `AT` 命令进行链路联调
- 点击“发送”后，状态栏先显示提交结果，再显示串口线程实际发送完成的字节数
- 勾选“HEX 发送”后，发送内容请按 `01 02 0A` 这类格式输入
- 勾选“HEX 显示”后，接收区将按十六进制显示
- 打开串口前可选择波特率、数据位、校验位、停止位
- 勾选“自动换行”后，接收区会按窗口宽度换行
- 定时发送可设置毫秒间隔，然后点击“开始定时发送”
- “发送历史”会保留本次运行期间成功发送过的内容
- 点击“保存命令集”可将当前“发送历史”保存到本地，并自定义命令集名称
- 命令集文件默认保存在 `data/command_sets/`，文件内容包含命令与“命令描述”
- 应用下次启动时会自动恢复上一次保存/使用的命令集
- 可通过“命令集”下拉列表快速切换并加载已保存命令集
- 保存命令集后，如继续发送新命令，程序会自动实时追加到当前命令集文件
- 日志区域会显示 `send/receive` 方向与时间，格式示例：`2025-03-02 16:28:30.123`
- 日志会自动落盘到 `log/YYYY-MM-DD.log`（同一天重复运行会继续追加）
- 普通发送后输入框内容不会被清空，程序重启后会自动恢复上次发送命令
- “send/receive 统计”会累计当前连接会话中的字节数，可手动清零
- “保存接收日志”会把当前接收区内容保存为 `UTF-8` 文本文件
- 发送区支持“命令描述”输入，用于记录当前命令用途（可选）
- “发送历史”会在命令后显示括号描述，例如：`AT+RST (重启模块)`；无描述则不显示括号
- 勾选“自动检测串口并重连”后，串口异常断开不会弹窗打断；程序会自动轮询并在端口恢复后直接重连

## 排查“发送后收不到数据”建议

- 先确认硬件链路：
  - 自发自收测试时，需将 `TX` 与 `RX` 正确短接，并确保 `GND` 共地
  - 两个设备互连时，请交叉连接 `TX->RX`、`RX->TX`
- 确认串口参数两端一致：波特率、数据位、校验位、停止位必须完全匹配
- 若设备协议要求结尾符，请在发送内容中补上（如 `\r`、`\n` 或 `\r\n`）
- 若使用 HEX 发送，必须是合法十六进制字节串（如 `01 03 00 00 00 02 C4 0B`）
- 接收线程采用阻塞读取（基于 `read(in_waiting or 1)`），避免空轮询导致的漏读与高 CPU 占用
- 观察状态栏：
  - 出现“发送请求已提交”表示 UI 已把数据放入发送队列
  - 出现“串口已发送 X 字节”表示串口线程已实际调用底层发送

## CI/CD（自动构建与发布）

项目新增 GitHub Actions 工作流：

- 普通 `push` / `pull_request`：自动安装依赖并执行 `python -m compileall` 语法检查
- 当推送 `v*` 标签（如 `v1.0.0`）时：在检查通过后自动创建 Release，并上传源码压缩包

工作流文件：`.github/workflows/ci-release.yml`
